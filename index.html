📡 Modulator — OOK-NRZ dan 4-PPM
OOK
Baris 1125 di sim_OOK() — bit dipetakan ke daya:
tx_signal = tx_bits * P_tx; % bit 1 → daya P_tx = 100 mW % bit 0 → daya 0 (tidak ada cahaya)
4-PPM
Baris 1167–1178 di sim_4PPM() — 2 bit dipetakan ke posisi slot:
% [0,0]→slot1, [0,1]→slot2, [1,0]→slot3, [1,1]→slot4 tx_ppm(i, tx_symbols(i)) = M * P_tx; % peak = 4×P_tx di slot aktif
PPM mengumpulkan energi di 1 slot, bukan menyebar di semua slot → lebih efisien daya.
Grafik

Kode: tx_shaped = conv(tx_optical, gauss_filter, 'same') baris 552
🔦 Laser diode — daya dan panjang gelombang
Ada di kode (parameter)
Didefinisikan di FASE 1 baris 38–40. Tidak ada proses fisik laser yang disimulasikan — hanya nilainya dipakai:
lambda = 1550e-9; % panjang gelombang (m) — ITU-R, eye-safe P_tx_dBm = 20; % daya pancar (dBm) — Gismalla et al. 2024 P_tx = 0.1; % = 100 mW (konversi dari dBm)
P_tx adalah amplitudo sinyal OOK dan puncak PPM. k_wave = 2π/λ dipakai untuk menghitung Rytov variance di turbulensi.

🔭 TX telescope — divergensi beam
Ada di kode (parameter)
θ_div = 2 mrad (baris 49) dipakai untuk menghitung radius beam Gaussian di jarak L:
w_z = D_rx/2 + theta_div * L; % contoh L=2km: w_z = 0.05 + 0.002×2000 = 4.05 meter % w_z ini masuk ke gen_point_err() untuk hitung A₀
Semakin jauh jarak, beam semakin melebar (w_z makin besar), sehingga A₀ (fraksi daya yang tertangkap lensa) makin kecil.

🌧 h_a — Redaman atmosfer (Beer-Lambert)
Ada di kode
Didefinisikan FASE 3 baris 184 sebagai anonymous function, dipanggil setiap loop. Bersifat deterministik — tidak random:
dBkm_to_per_m = @(x) (x / 4.343) / 1000; % konversi satuan calc_h_atm = @(beta_dBkm, L_meter) exp(-dBkm_to_per_m(beta_dBkm) .* L_meter); % dipanggil di setiap loop: h_a = calc_h_atm(beta_total(s), L); % baris 254
h_a dihitung sekali per kombinasi (cuaca, jarak), tidak di-generate ulang tiap iterasi MC karena deterministik.
Grafik
G1 langsung dari h_a G4 β = sumbu X G2 G3 G5 via h_total
G1 = hanya h_a, tidak ada h_t dan h_p, tidak ada Monte Carlo. G4 = β (penyebab h_a) yang divariasikan di sumbu X dengan garis vertikal posisi tiap cuaca Indonesia.
🌀 h_t — Turbulensi atmosfer (Log-Normal)
Ada di kode
Fungsi gen_turb_LN() baris 1073–1083. Dipanggil setiap iterasi MC — nilainya selalu baru:
function h_t = gen_turb_LN(Cn2, k_wave, L, N_samples) sigma_R2 = 1.23 * Cn2 * k_wave^(7/6) * L^(11/6); % Rytov variance sigma_X2 = sigma_R2 / 4; X = sqrt(sigma_X2) * randn(1, N_samples); % ← RANDOM (randn) h_t = exp(2 * (X - sigma_X2)); % log-normal end
E[h_t] ≈ 1 — rata-rata tidak menambah loss. Tapi per-realisasi bisa lebih besar atau lebih kecil (fluktuasi = scintillation).
Grafik
G5 Cn² = sumbu X G2 G3 via h_total G7(c) via h_total
G5 mengisolasi efek h_t saja — hanya Cn² yang berubah, cuaca cerah, jarak tetap 2 km. Turbulensi baru signifikan di atas Cn² ≈ 10⁻¹⁴.
🎯 h_p — Pointing error (Farid & Hranilovic 2007)
Ada di kode
Fungsi gen_point_err() baris 1094–1109. Dipanggil setiap iterasi MC:
function h_p = gen_point_err(sigma_jitter, w_z, a_rx, N_samples) v = sqrt(pi) * a_rx / (sqrt(2) * w_z); A0 = (erf(v))^2; % fraksi daya saat beam center w_eq2 = w_z^2 * sqrt(pi)*erf(v) / (2*v*exp(-v^2)); r_d = sigma_jitter * sqrt(-2*log(rand(1,N))); % ← RANDOM Rayleigh h_p = A0 * exp(-2 * r_d.^2 / w_eq2); end
sigma_jitter = 0.1 m dari Dabiri & Sadough (2019) untuk UAV hovering. r_d = pergeseran radial beam akibat getaran UAV.
Grafik
G2 G3 via h_total G8 G9 spreading distribusi G10 G11 kualitas eye
h_p menyumbang fading acak ke h_total. Karena random, efeknya memperlebar distribusi di G8/G9 dan menyempitkan bukaan mata di G10/G11.
🔭 RX telescope — menangkap cahaya
Parameter saja
Tidak ada proses fisik teleskop yang disimulasikan. Efeknya direpresentasikan oleh parameter D_rx baris 45:
D_rx = 0.10; % diameter aperture = 10 cm a_rx = D_rx / 2; % radius = 5 cm theta_FoV = 10e-3; % field of view = 10 mrad
a_rx masuk ke gen_point_err() untuk menghitung A₀ = [erf(v)]² — yaitu berapa fraksi daya yang bisa tertangkap lensa saat beam tepat di center aperture.

🔊 TIA + BPF — amplifier, noise, dan filter
Ada di kode — dua bagian
Bagian 1 — Model noise dari TIA (FASE 1 baris 59–65):
sigma2_thermal = 4 * 1.381e-23 * T_rx * B_noise / R_L; % R_L = 1000 Ω = impedansi TIA % T_rx = 300 K = suhu tropis Indonesia (merepresentasikan noise TIA) % B_noise = Rb/2 = 500 MHz = bandwidth BPF N0 = sigma2_thermal + sigma2_shot + sigma2_bg;
Bagian 2 — Noise AWGN ditambahkan di dalam sim_OOK() baris 1134:
noise = sqrt(N0) * randn(1, N_bits); % ← RANDOM noise rx_signal = rx_current + noise;
Bagian 3 — Filter penerima (BPF/matched filter) di Grafik 7 baris 558:
rx_filtered = conv(rx_noisy, gauss_filter, 'same'); % matched filter Gaussian mengurangi noise di luar bandwidth
Thermal noise dominan >95% dari N0 total karena R_L=1000Ω dan T=300K.

Perbedaan G7(d) dan G7(e) menunjukkan efek filter — sinyal di (e) lebih halus. Threshold terlihat di kedua subplot.
⚖️ Demodulator — keputusan bit (threshold / ML)
Ada di kode
OOK — threshold detection, baris 1138–1139 di sim_OOK():
threshold = (R_det * P_tx * h_channel) / 2; % threshold = tepat di tengah antara level "0" dan level "1" % adaptif: bergantung h_channel yang berubah tiap realisasi rx_bits = double(rx_signal > threshold); % sinyal > threshold → putuskan bit "1" % sinyal ≤ threshold → putuskan bit "0"
4-PPM — ML detection, baris 1184:
[~, rx_symbols] = max(rx_ppm, [], 2); % pilih slot (dari 4) yang punya arus terbesar % ini Maximum Likelihood detection → lebih tahan fading
Grafik
G7(d)(e) garis threshold G8 area overlap = error G9 slot aktif vs kosong G10 G11 bukaan mata
Garis threshold terlihat jelas di G7(d) dan (e). G8 menunjukkan mengapa threshold gagal saat cuaca buruk — distribusi bit "0" dan "1" bertumpuk (overlap).
